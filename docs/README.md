## Быстрый старт

### Базовый URL API
```
http://localhost:8000/api/v1
```

### Аутентификация
Большинство endpoints требуют JWT токен в заголовке:
```
Authorization: Bearer <access_token>
```

Получить токен можно через:
```bash
POST /api/v1/auth/login
```

### Пример запроса
```bash
curl -X GET "http://localhost:8000/api/v1/orders/" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Структура системы

### Основные модули

1. **Аутентификация** (`/api/v1/auth`)
   - Вход в систему
   - Обновление токенов
   - Регистрация пользователей

2. **Пользователи** (`/api/v1/users`)
   - Управление пользователями
   - Назначение ролей

3. **Транспортные средства** (`/api/v1/vehicles`)
   - Регистрация автомобилей клиентов
   - Хранение данных владельцев

4. **Заказ-наряды** (`/api/v1/orders`)
   - Создание и управление заказ-нарядами
   - Назначение механиков
   - Завершение работ

5. **Работы и запчасти** (`/api/v1/works`, `/api/v1/parts`)
   - Каталог видов работ
   - Каталог запчастей

6. **Склад** (`/api/v1/warehouse`)
   - Управление остатками
   - Приход и расход товаров
   - Уведомления о низких остатках

7. **Сотрудники** (`/api/v1/employees`)
   - Управление персоналом
   - Данные сотрудников

8. **Зарплата** (`/api/v1/salary`)
   - Расчет зарплаты
   - Учет выполненных заказ-нарядов
   - Отметка о выплате

9. **Платежи** (`/api/v1/payments`)
   - Оплата заказ-нарядов
   - Интеграция с ЮKassa

10. **Интеграции** (`/api/v1/integrations`)
    - Проверка ГИБДД
    - Поиск запчастей у поставщиков
    - Заказы у поставщиков

## Роли пользователей

### ADMIN (Администратор)
- Полный доступ ко всем функциям системы
- Управление пользователями
- Все операции менеджера, механика и бухгалтера

### MANAGER (Менеджер)
- Создание и управление заказ-нарядами
- Управление складом
- Управление сотрудниками
- Создание платежей
- Работа с транспортными средствами

### MECHANIC (Механик)
- Просмотр назначенных заказ-нарядов
- Завершение заказ-нарядов
- Просмотр работ и запчастей
- Просмотр информации о транспортных средствах

### ACCOUNTANT (Бухгалтер)
- Расчет зарплаты сотрудников
- Отметка о выплате зарплаты
- Просмотр платежей
- Работа с финансовой отчетностью

## Ключевые концепции

### Заказ-наряд
Центральная сущность системы, которая связывает:
- Клиента (через транспортное средство)
- Работы и запчасти
- Механика и менеджера
- Платежи
- Складские операции

**Статусы:**
- `NEW` - Создан, ожидает назначения механика
- `IN_PROGRESS` - В работе
- `COMPLETED` - Завершен (запчасти списаны со склада)
- `CANCELLED` - Отменен

### Склад
Система отслеживает остатки запчастей:
- Каждая запчасть имеет одну позицию на складе
- Приход товара увеличивает остаток
- Расход происходит автоматически при завершении заказ-наряда
- Система уведомляет о низких остатках

### Зарплата
Автоматический расчет на основе:
- Базовой зарплаты сотрудника
- Количества выполненных заказ-нарядов за период
- Бонусов (5% от базовой за каждый заказ)
- Штрафов (пока не реализовано)

## Примеры использования

### Создание заказ-наряда

```bash
POST /api/v1/orders/
Authorization: Bearer <token>
Content-Type: application/json

{
  "vehicle_id": 1,
  "mechanic_id": 3,
  "order_works": [
    {
      "work_id": 1,
      "quantity": 2,
      "price": "5000.00"
    }
  ],
  "order_parts": [
    {
      "part_id": 1,
      "quantity": 1,
      "price": "5000.00"
    }
  ]
}
```

### Завершение заказ-наряда

```bash
POST /api/v1/orders/1/complete
Authorization: Bearer <token>
```

При завершении автоматически:
- Списываются запчасти со склада
- Создаются транзакции расхода
- Обновляется статус на `COMPLETED`

### Расчет зарплаты

```bash
POST /api/v1/salary/calculate
Authorization: Bearer <token>
Content-Type: application/json

{
  "employee_id": 1,
  "period_start": "2024-01-01",
  "period_end": "2024-01-31"
}
```

Система автоматически:
- Берет базовую зарплату сотрудника
- Подсчитывает выполненные заказ-наряды за период
- Рассчитывает бонусы
- Создает запись расчета

## Технические детали

### Формат данных
- **Даты**: `YYYY-MM-DD` (например, `2024-01-01`)
- **Дата и время**: ISO 8601 `YYYY-MM-DDTHH:MM:SSZ`
- **Денежные суммы**: Строки с двумя знаками после запятой (например, `"15000.00"`)

### Пагинация
Большинство endpoints списков поддерживают пагинацию:
- `skip` - количество записей для пропуска (по умолчанию 0)
- `limit` - максимальное количество записей (по умолчанию 100)

### Обработка ошибок
API возвращает стандартные HTTP статусы:
- `200 OK` - Успешный запрос
- `201 Created` - Ресурс создан
- `400 Bad Request` - Неверные параметры
- `401 Unauthorized` - Требуется авторизация
- `403 Forbidden` - Недостаточно прав
- `404 Not Found` - Ресурс не найден
- `500 Internal Server Error` - Ошибка сервера
