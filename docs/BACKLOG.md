# Бэклог проекта AutoService Management

## Высокий приоритет

### Интеграция с MotorData API (нормы времени)
- **Описание**: Интеграция с [MotorData](https://motordata.net/) для получения профессионального каталога работ с нормо-часами, привязанными к конкретным маркам и моделям автомобилей.
- **Зачем**: Базовый каталог (~250 работ) покрывает потребности небольших сервисов, но для крупных автосервисов и SaaS-продукта нужен профессиональный каталог с 500 000+ нормативов.
- **Что нужно сделать**:
  - [ ] Связаться с MotorData по поводу OEM/партнёрской программы для SaaS
  - [ ] Изучить документацию API MotorData
  - [ ] Добавить модель `Integration` / настройку API-ключа MotorData в settings
  - [ ] Реализовать сервис `motordata_service.py` для запросов к API
  - [ ] API эндпоинт: поиск работ по марке/модели через MotorData
  - [ ] Фронтенд: при создании наряда — выбор марки/модели → подгрузка работ из MotorData
  - [ ] Fallback на локальный каталог, если MotorData недоступен или не настроен
- **Тарифы MotorData** (на март 2026):
  - Старт: ~1 390 ₽/мес, до 1 000 запросов
  - Оптимум (5 юзеров): ~332 ₽/мес за юзера, до 42 000 запросов
  - Оплата разовая за период, не рекуррентная
- **Варианты для SaaS**:
  - Каждый клиент подключает свой ключ MotorData
  - Или партнёрское соглашение для перепродажи

## Средний приоритет

### Скрипт инициализации системных категорий кассы
- **Описание**: Создать скрипт `backend/scripts/seed_cashflow_categories.py`, который при запуске автоматически создаёт все системные категории, необходимые для работы модуля кассы.
- **Зачем**: При развёртывании приложения системные категории (`is_system=true`) должны существовать с первого запуска — без них невозможна автоматическая привязка операций при оплате заказов и выплате зарплат. Сейчас они создаются вручную.
- **Системные категории** (согласно БТ-05, TC-08):
  - "Оплата заказа" — `transaction_type: income`, `is_system: true`
  - "Выплата зарплаты" — `transaction_type: expense`, `is_system: true`
  - "Начальный остаток" — `transaction_type: income`, `is_system: true`
  - "Перевод между счетами" — `transaction_type: transfer`, `is_system: true`
- **Что нужно сделать**:
  - [ ] Написать `backend/scripts/seed_cashflow_categories.py` по аналогии с `seed_works_catalog.py` (проверка дублей, idempotent запуск)
  - [ ] Добавить скрипт в `docs/SCRIPTS.md` в раздел "Порядок запуска при первом развёртывании"
  - [ ] Рассмотреть вариант автозапуска через `startup` event FastAPI (чтобы системные категории создавались при старте приложения, если их нет)
- **Заметка**: уже есть аналогичные скрипты — `seed_car_catalog.py`, `seed_works_catalog.py`; паттерн понятен.
