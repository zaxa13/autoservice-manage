# AutoService SaaS — Бэклог разработки

> **Дата создания:** 2026-04-03
> **Методология:** Инкрементальная разработка. Каждый этап — завершённая, тестируемая фича.
> **Принцип:** Сделали → протестировали → если ОК → следующая фича.

---

## Разделение по репозиториям

| Репозиторий | Что | Бэклог |
|-------------|-----|--------|
| **autoservice-management** (этот) | Tenant App — само приложение автосервиса | Этапы 0, 1, 6 (частично) |
| **autoservice-platform** | Platform API, лендинг, ЛК владельца, админка, провижининг | Этапы 2, 3, 4, 5 |

> Этапы 0 и 1 — задачи этого репозитория.
> Этапы 2-5 — задачи репозитория `autoservice-platform` (см. бэклог там).
> Этап 6 — пост-MVP улучшения, распределены по обоим репозиториям.

---

## Как читать этот бэклог

- **Этап** — крупный блок работ (эпик), объединённый бизнес-целью
- **Фича** — конкретная задача внутри этапа, которую можно сделать и протестировать отдельно
- **Приоритет:** 🔴 Критично | 🟡 Важно | 🟢 Желательно
- **Статус:** ⬜ Не начато | 🔧 В работе | ✅ Готово | ⏸ Отложено
- **Зависимости:** если фича зависит от другой — указано явно

---

## Общая дорожная карта (верхний уровень)

```
[autoservice-management]
  Этап 0: Критические пробелы текущего приложения     ← СНАЧАЛА ЭТО
  Этап 1: SaaS-адаптация (ENV, лимиты, wizard)

[autoservice-platform]
  Этап 2: Platform API (регистрация, биллинг)
  Этап 3: Лендинг и личный кабинет владельца
  Этап 4: Провижининг и инфраструктура
  Этап 5: Админ-панель платформы

[оба репозитория]
  Этап 6: Пост-MVP улучшения (проценка, SMS, платежи тенанта)
```

> **Почему этап 0 первый?** Бизнес-план показывает готовность приложения ~70%.
> Критические пробелы (печать, мобильная адаптация, зарплата) блокируют продажу.
> Нет смысла строить платформу вокруг продукта, который клиент не купит.
