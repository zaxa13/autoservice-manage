# Создание первого администратора

## Разница между User и Employee

### User (Пользователь)
- **Назначение**: Учетная запись для входа в систему
- **Поля**: 
  - `username` - имя пользователя для входа
  - `email` - email (уникальный)
  - `password_hash` - хеш пароля
  - `role` - роль в системе (admin, manager, mechanic, accountant)
  - `employee_id` - связь с сотрудником (опционально)
  - `is_active` - активен ли пользователь

### Employee (Сотрудник)
- **Назначение**: Карточка сотрудника с личными данными
- **Поля**:
  - `full_name` - полное имя сотрудника
  - `position` - должность (admin, manager, mechanic)
  - `phone` - телефон
  - `email` - email
  - `hire_date` - дата приема на работу
  - `salary_base` - базовая зарплата
  - `is_active` - активен ли сотрудник

### Связь между User и Employee
- User может быть связан с Employee через `employee_id` (опционально)
- Можно иметь User без Employee (например, для внешних пользователей)
- Можно иметь Employee без User (например, для сотрудников без доступа к системе)

## Создание первого администратора

### Способ 1: Через API (рекомендуется)

Если в системе еще нет пользователей, первый пользователь автоматически создается как администратор **без требования авторизации**.

**Запрос:**
```bash
POST /api/v1/auth/register
Content-Type: application/json

{
  "username": "admin",
  "email": "admin@example.com",
  "password": "your_password",
  "role": "admin"
}
```

**Важно:** 
- Если пользователей в системе нет, поле `role` игнорируется - пользователь всегда создается как `admin`
- Если пользователи уже есть, требуется авторизация администратора

### Способ 2: Через скрипт

```bash
cd backend
python scripts/create_admin.py
```

Скрипт запросит:
- Username
- Email
- Password

И создаст первого администратора, если в системе еще нет пользователей.

## После создания первого администратора

1. Войдите в систему через `/api/v1/auth/login`
2. Используйте полученный токен для создания других пользователей через `/api/v1/auth/register`
3. При необходимости создайте Employee и свяжите его с User через `employee_id`
